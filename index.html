MacOS-Btn Description and usage

This page displays the methods for displaying the macos buttons

Static HTML

These buttons are statically placed in the html file containing: An outer html div with the ID of the button (also some left-spacing to look nicer), an inner-outer div layer that creates a macos looking gradient border (css is just a 1px pad with the gradient background), and an inner-content div that contains the content of the button - and css to make it look nice. You can disable custom font and cursor in the macos-btn.css file.

Dynamic HTML / jQuery appended class instances

These buttons are dynamically generated with the short code:

$('body').append(new macosBtn("Main btn jQuery append", 1, "#", "id").getHtml());
$('body').append(new macosBtn("Secondary btn jQuery append", 2, "#", "id").getHtml());

The class constructor's argument descriptions are:

var btn = new macosBtn(String txt, Int type, String href, String id);

String txt: The label that the button will display
Int type: The type type of button style (1 or 2 by default, add more css styles and options with the cases)
String href: The  href to redirect the user on click
String id: The Id for the button to use as a selector


Additional info

If you don't know which to use, a short explanation: Static HTML buttons should be used on webpages and template webpages that do not have dynamic content, e.g. Homepage, pages where the user is not required to have javascript enabled. Dynamic HTML buttons should be used with dynamically accessed interfaces, such as a pop-up modal, dynamic content, and any screens/content which should require the used of javascript. It may be worth implementing a contingency plan for user's that do not have js enabled, so they acrtually have buttons to click - or implement an entirely different (non-js) method of displaying the content.
